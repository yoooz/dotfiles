# jot

yoooz_vault の今日のデイリーノートに Thino 形式のつぶやきを1行追加するスキル。

## いつ使うか（中粒度のトリガー）

Claude が能動的に呼び出すこと。具体的には以下のタイミング:

- 機能の実装が一区切りついた
- `git commit` を作成した
- PR を作成した
- 調査・デバッグで何か判明した
- 設計方針が確定した
- ユーザーが明示的に「つぶやいて」「メモして」「jot」と指示した

逆に使わない場面:

- ファイル1つを編集しただけ（粒度が小さすぎる）
- ユーザーとの単なる会話の応答
- エラーで何も進まなかった時

## Usage

```bash
<SKILL_DIR>/scripts/jot.sh "メッセージ"
```

`<SKILL_DIR>` は本 SKILL.md が置かれているディレクトリ（`~/.claude/skills/jot`）を指す。呼び出し側は実行時に絶対パスへ展開すること。

### メッセージの書き方

- 1行・短文（後日、日報作成時に見返して意味が思い出せる粒度）
- 過去形・体言止めどちらでもよい。例:
  - `"jobcan-cli に jot skill を追加"`
  - `"prefetch の競合バグの原因を特定: cache key 衝突"`
  - `"PR #123 作成"`

### 動作

1. 現在時刻を `HH:MM:SS` 形式で取得（OS時刻）
2. `~/yoooz_vault/daily_notes/YYYY/MM/YYYY-MM-DD.md` を対象ファイルとする
3. ファイルが存在しなければ stderr にエラーを出し `exit 1`
4. ファイル末尾に `- HH:MM:SS メッセージ` を追記
5. 追記した内容を stdout に出力

### 環境変数

- `YOOOZ_VAULT_DIR`: vault のパスを上書きしたい場合に指定（デフォルト `$HOME/yoooz_vault`）

## エラー時の振る舞い

- **今日のデイリーノートが存在しない**: スキップせず、Claude はユーザーに「今日のデイリーノートが存在しないので jot できなかった」と一言報告すること
- **書き込み失敗**: stderr の内容をユーザーに伝える

## Out of scope

- 過去日付のデイリーノートへの追記
- 既存のつぶやきの編集・削除
- デイリーノートそのものの自動生成
- 「つぶやき」セクションが末尾にない場合の挿入位置調整（現状の vault フォーマットでは常に末尾にある前提）
