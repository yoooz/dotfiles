# session_retrospective

今回のセッションを5つの観点で振り返り、結果をマークダウンとして `~/yoooz_vault/ai_session_retrospectives/` に保存するスキル。

なお、`~/yoooz_vault/振り返り/` は **AI/Claudeセッションとは無関係な、普段の振り返り用途** で別管理されている。混同しないよう、本スキルは必ず `ai_session_retrospectives/` を使うこと。

## 目的

AIをもっとうまく使っていくために、今回のセッションから **継続すべきこと** と **改善すべきこと** をキャッチアップする。

## いつ使うか

ユーザーが以下のように明示的に指示した時:

- 「振り返りして」「ふりかえり」「セッションを振り返って」
- 「retrospective」「リフレクション」
- `/session_retrospective` と直接呼び出し

逆に使わない場面:

- セッションがほとんど進んでいない時（振り返る材料が無い）
- 1コマンドだけ実行して終わった時

## 振り返りの観点

以下の5つの観点でセッション全体を振り返る:

1. **タスク完了までの全体の流れ**
   - どんなステップで進めたか
   - 手戻りや遠回りはあったか
2. **タスクの粒度に関すること**
   - タスクが大きすぎなかったか、小さすぎなかったか
   - 分割の仕方は妥当だったか
3. **タスクの事前情報に関すること**
   - 必要な情報は揃っていたか
   - どんな情報が不足していて確認が必要だったか
4. **途中のレビューに関すること**
   - ユーザーからのレビュー・フィードバックは適切なタイミングで入ったか
   - レビュー結果を踏まえた軌道修正はうまくいったか
5. **その他継続/改善すべきこと**
   - 上記4観点に当てはまらない、特筆すべき良かった点・改善点

## 出力ファイル

### 保存先

`~/yoooz_vault/ai_session_retrospectives/<YYYY-MM-DD> <タスク概要>.md`

- `YYYY-MM-DD` は **`date '+%Y-%m-%d'` で取得した今日の日付**
- `<タスク概要>` は今回のセッションの内容を一目で思い出せる短いタイトル（例: `hn-reader skill化`, `jobcan-cli CI修正`）
- 拡張子は `.md`

### 出力フォーマット

```markdown
# <YYYY-MM-DD> <タスク概要>

## セッション概要
<1-3行で何をやったかのサマリ>

## 1. タスク完了までの全体の流れ
- 良かった点:
- 改善点:

## 2. タスクの粒度
- 良かった点:
- 改善点:

## 3. タスクの事前情報
- 良かった点:
- 改善点:

## 4. 途中のレビュー
- 良かった点:
- 改善点:

## 5. その他継続/改善すべきこと
-

## 次回以降のアクション
- 継続すべきこと:
- 改善すべきこと:
```

## 注意点

- **技術的な内容は抽象化して、一般的な技術知識として蓄積できる形にする**
  - 例: 「`hn-reader/SKILL.md` のWebFetchが…」ではなく「外部ページのHTML構造に依存するスキルは取得後の構造化抽出が崩れるリスクがある」
- 個別プロジェクトの固有名詞は最小限にとどめる
- 振り返りは「事実 → 解釈 → 次のアクション」の順で整理する

## エラー時の振る舞い

- **`~/yoooz_vault/ai_session_retrospectives/` ディレクトリが存在しない**: 作成せず、ユーザーに「振り返りディレクトリが見つからなかった」と報告
- **ファイル名がすでに存在する**: 上書きせず、末尾に連番（` 2.md`, ` 3.md`）を付けて新規作成

## Out of scope

- 過去のセッションの振り返り（あくまで「今回のセッション」のみ）
- 月次・年次まとめへの統合（既存の `YYYY-MM.md` への追記は別作業）
- 振り返り内容に基づいて memory への自動保存（明示的に指示された場合を除く）
